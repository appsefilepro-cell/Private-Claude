## üìä CLIENT & CASE MANAGEMENT SYSTEM

### **Database Structure (Airtable):**

**5 Tables (Relational Database):**

1. **Clients Table** - Store ONLY client information
   - client_id (CLI-xxx), name, type, email, phone, status, tags, notes

2. **Cases Table** - Store ONLY case information
   - case_id (CASE-xxx), case_number, type, status, court, plaintiff, defendant, dates, documents

3. **Partnerships Table** - Connect multiple clients WITHOUT mixing details
   - partnership_id (PART-xxx), name, type, linked clients, dates, agreements

4. **Situations Table** - Describe situations with multiple clients/cases
   - situation_id (SIT-xxx), description, type, dates, status, outcome

5. **Relationships Table** - Map ALL cross-references
   - relationship_id (REL-xxx), entity_1, entity_2, relationship_type, strength, notes

### **Prevention of Mix-Ups:**

‚úÖ **NEVER** store client details in case records
‚úÖ **NEVER** store case details in client records
‚úÖ **ALWAYS** use unique IDs for each entity
‚úÖ **ALWAYS** create relationship records for connections
‚úÖ Use Airtable linked records for visual connections
‚úÖ Monthly data validation by Agent 5.0 Legal Division
